# NotLettersHUB

Desktop-приложение на Python и Flet для получения и просмотра писем с нескольких аккаунтов через API NotLetters.

## Возможности

- Работа с несколькими аккаунтами.
- Добавление, редактирование, удаление и импорт аккаунтов из TXT.
- Глобальный API-ключ и токены на уровне отдельных аккаунтов.
- Поиск по письмам и фильтры.
- Ручное обновление выбранного аккаунта и обновление всех аккаунтов.
- Пакетное автообновление с лимитом по количеству аккаунтов и паузой между пачками.
- Просмотр text и HTML версии письма.
- Локальное кэширование писем в памяти.
- Безопасная обработка сетевых и API-ошибок без падения UI.

## Структура проекта

- `main.py` — точка входа.
- `models.py` — dataclass-модели.
- `api_client.py` — работа с API NotLetters.
- `storage.py` — локальное хранилище и импорт аккаунтов.
- `ui/app.py` — основная логика интерфейса.
- `ui/dialogs.py` — диалоги и формы.
- `build_exe.ps1` — сборка desktop-приложения через PyInstaller.
- `NotLettersDesktop.spec` — spec-файл PyInstaller.

## Хранение данных

Пользовательские данные не хранятся рядом с `.exe`.

Приложение сохраняет данные в:

`%LOCALAPPDATA%\NotLettersHUB`

Там создаются:

- `accounts.json`
- `app_settings.json`

Это удобно для установленной версии приложения и не требует записи в папку установки.

## Запуск из исходников

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

## Сборка desktop-приложения

```powershell
.\build_exe.ps1
```

Готовая сборка появится в папке `dist\NotLettersDesktop`.

## Удаление приложения

Deinstall удаляет:

- папку установки приложения;
- ярлыки;
- запись в списке установленных программ.

При обычном uninstall установщик дополнительно спрашивает:

`Delete local data and settings too?`

Если подтвердить, будет удалена папка:

`%LOCALAPPDATA%\NotLettersHUB`

При silent-uninstall во время переустановки локальные данные сохраняются.
